# Project Constitution

## Purpose
The AI-Based Idea Validation Engine helps founders validate startup ideas using AI-powered analysis across market, competitor, feasibility, financial, risk, scoring, and reporting dimensions.

## Principles

### 1. Spec-Driven Development
All major features must begin with a written specification that captures user goals, functional requirements, non-functional requirements, acceptance criteria, and risks before implementation starts.

### 2. Testable Outcomes
Every feature specification must define acceptance criteria that can be verified through automated tests, manual checks, or both.

### 3. Security and Privacy
The project must protect user-submitted idea data, avoid committing secrets, and keep static analysis and dependency audit tooling in the development workflow.

### 4. Documentation
User-facing behavior, setup changes, and operational workflows must be documented when they change.

### 5. Quality Standards
Code changes must remain compatible with the repository quality toolchain:
- Ruff
- Flake8
- Pylint
- Mypy
- Bandit
- Vulture
